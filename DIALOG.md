**CURSOR**:
Created CARMEN repository based on existing agent patterns (AGAPE, ERIS, METIS, THALIA, ROSE, THOTH):

**Unified Workflow Design**:
1. **Stage 1 - AGAPE**: Analyze love as ACTION (verb, not feeling)
   - What loving actions can be taken? What helps NOW?
   - Behavioral markers: teach, help, encourage, patient, kind, hopeful

2. **Stage 2 - ERIS**: Identify ROOT CAUSES of conflicts
   - Not "you're angry" but "THIS is causing the anger"
   - Knowledge gaps, misunderstandings, ambiguity, constraints

3. **Stage 3 - METIS**: Run empathy through comparison
   - What SHOULD be vs what IS
   - Understanding the gap between reality and ideal
   - Not sympathy, but empathy (understanding the gap)

4. **Stage 4 - THALIA + ROSE**: See what could be FUNNY
   - Not jokes, but genuine humor from context
   - Cultural context checks via ROSE
   - Slang translation, cultural awareness

5. **Stage 5 - THOTH**: Verify truth
   - Categorize claims: VERIFIED/ANECDOTAL/THEORETICAL/UNVERIFIED
   - Source tracing, zero-bullshit policy

**Key Concepts**:
- **Single-Pass Processing**: All 5 stages process internally, one unified response
- **Efficiency**: One API call instead of 6-7 agent chain calls
- **Experimental**: Proof of concept, not production-ready yet
- **Unified Response**: Combines all emotional intelligence insights into one reply

**Problem Solved**:
- **Before**: Send message to AGAPE → ERIS → METIS → THALIA → ROSE → THOTH (6-7 calls)
- **After**: Send message to CARMEN once → All 5 stages process internally → One unified response

**Key Learning**: Unified emotional intelligence is possible, but needs testing. The tradeoff is speed vs. depth—single-pass may be faster but may sacrifice the depth that comes from specialized agents working independently.

**Outcome**:
- CARMEN repository created
- Unified workflow documented
- 5-stage processing pipeline designed
- Experimental status clearly marked
- Ready for implementation (next step)
